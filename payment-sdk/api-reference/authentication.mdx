## Bearer Token Authentication

All Grain API requests require a valid API key passed as a Bearer token in the `Authorization` header.

```bash
curl -X GET https://api.grain.com/v1/payments \
  -H "Authorization: Bearer sk_live_your_secret_key"
```

```typescript
const response = await fetch("https://api.grain.com/v1/payments", {
  headers: {
    Authorization: `Bearer ${process.env.GRAIN_SECRET_KEY}`,
    "Content-Type": "application/json",
  },
});
```

## API Key Types

Grain issues two types of API keys per environment.

| Key Type | Prefix | Use | Permissions |
| --- | --- | --- | --- |
| Publishable key | `pk_live_` / `pk_test_` | Client-side (browser) | Launch payment UI, read session status |
| Secret key | `sk_live_` / `sk_test_` | Server-side only | Create sessions, read payments, manage webhooks, full access |

<Warning>
  Never expose your **secret key** in client-side code, public repositories, or frontend bundles. Secret keys have full access to your Grain account, including payment data and settlement configuration. If you suspect a key has been compromised, rotate it immediately in the dashboard.
</Warning>

## Include Your Key in Requests

### Server-Side (Secret Key)

```typescript
import { GrainAPI } from "@grain-pay/sdk/server";

const grain = new GrainAPI({
  secretKey: process.env.GRAIN_SECRET_KEY, // sk_live_...
});
```

### Client-Side (Publishable Key)

```typescript
import { Grain } from "@grain-pay/sdk";

const grain = new Grain({
  apiKey: process.env.NEXT_PUBLIC_GRAIN_KEY, // pk_live_...
  environment: "production",
});
```

## Rotate API Keys

<Steps>
  <Step title="Generate a new key">
    In the [Grain Dashboard](https://dashboard.grain.com), go to **Settings > API Keys** and click **Generate New Key**.
  </Step>

  <Step title="Update your application">
    Replace the old key in your environment variables and deploy.
  </Step>

  <Step title="Revoke the old key">
    Once your application is running with the new key, click **Revoke** on the old key in the dashboard.
  </Step>
</Steps>

<Note>
  Revoking a key is immediate. Any in-flight requests using the revoked key will fail with a `401 Unauthorized` error.
</Note>
