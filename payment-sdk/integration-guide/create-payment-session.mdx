## Server-Side: Create a Session

Create payment sessions from your backend using your **secret key**. This returns a session ID that your client uses to launch the payment UI.

```typescript
// Server-side (Node.js / Express)
import { GrainAPI } from "@grain-pay/sdk/server";

const grain = new GrainAPI({ secretKey: "sk_test_your_secret_key" });

app.post("/api/create-payment", async (req, res) => {
  const session = await grain.paymentSessions.create({
    amount: "100.00",
    currency: "USDC",
    chain: "ethereum",
    expiresIn: 1800, // 30 minutes
    metadata: {
      orderId: req.body.orderId,
      customerId: req.body.customerId,
    },
    successUrl: "https://yourapp.com/order/success",
    cancelUrl: "https://yourapp.com/order/cancel",
  });

  res.json({ sessionId: session.id });
});
```

### Request Parameters

| Parameter | Type | Required | Description |
| --- | --- | --- | --- |
| `amount` | `string` | Yes | Payment amount as a decimal string |
| `currency` | `string` | Yes | Token symbol (`USDC`, `USDT`, `ETH`, `SOL`, etc.) |
| `chain` | `string` | Yes | Target blockchain (`ethereum`, `polygon`, `arbitrum`, `base`, `solana`) |
| `expiresIn` | `number` | No | Session TTL in seconds (default: `1800`) |
| `metadata` | `object` | No | Arbitrary key-value pairs attached to the session |
| `successUrl` | `string` | No | Redirect URL after successful payment |
| `cancelUrl` | `string` | No | Redirect URL if the customer cancels |
| `receiverAddress` | `string` | No | Override the default settlement address |

### Response

```json
{
  "id": "ps_abc123def456",
  "status": "created",
  "amount": "100.00",
  "currency": "USDC",
  "chain": "ethereum",
  "expiresAt": "2026-03-20T12:30:00Z",
  "paymentUrl": "https://pay.grain.com/ps_abc123def456",
  "metadata": {
    "orderId": "order_12345",
    "customerId": "cust_67890"
  }
}
```

## Client-Side: Launch the Payment UI

Pass the session ID to the SDK on the client to open the payment modal.

<CodeGroup>
```tsx React
import { useGrain } from "@grain-pay/sdk/react";

function CheckoutButton({ orderId }: { orderId: string }) {
  const { launchPayment } = useGrain();

  const handleCheckout = async () => {
    const res = await fetch("/api/create-payment", {
      method: "POST",
      headers: { "Content-Type": "application/json" },
      body: JSON.stringify({ orderId }),
    });
    const { sessionId } = await res.json();

    launchPayment(sessionId, {
      onSuccess: (payment) => {
        window.location.href = `/order/success?id=${payment.id}`;
      },
      onError: (error) => {
        console.error("Payment failed:", error.message);
      },
    });
  };

  return <button onClick={handleCheckout}>Pay with Crypto</button>;
}
```

```typescript Vanilla JS
const grain = new Grain({ apiKey: "pk_test_your_key", environment: "sandbox" });

document.getElementById("pay-btn").addEventListener("click", async () => {
  const res = await fetch("/api/create-payment", {
    method: "POST",
    headers: { "Content-Type": "application/json" },
    body: JSON.stringify({ orderId: "order_12345" }),
  });
  const { sessionId } = await res.json();

  grain.launchPaymentUI(sessionId, {
    onSuccess: (payment) => {
      window.location.href = `/order/success?id=${payment.id}`;
    },
    onError: (error) => {
      console.error("Payment failed:", error.message);
    },
  });
});
```
</CodeGroup>

<Tip>
  You can also redirect customers to the hosted `paymentUrl` returned in the session response instead of using the embedded modal.
</Tip>
