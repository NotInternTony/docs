## Why Use Webhooks

Webhooks notify your server in real time when events occur in your Grain account — such as a payment being completed, a refund being issued, or a payout being sent. Instead of polling the API for status changes, your application receives an HTTP POST request with the event data as soon as it happens.

## Register a Webhook Endpoint

<Steps>
  <Step title="Create an endpoint on your server">
    Set up a publicly accessible HTTPS endpoint on your server that can receive POST requests. The endpoint should return a `200` status code to acknowledge receipt.

    ```javascript
    // Example: Express.js webhook handler
    app.post("/webhooks/grain", (req, res) => {
      const event = req.body;

      switch (event.type) {
        case "payment.completed":
          // Fulfill the order
          break;
        case "payment.failed":
          // Notify the customer
          break;
        case "refund.completed":
          // Update your records
          break;
      }

      res.status(200).json({ received: true });
    });
    ```
  </Step>

  <Step title="Register the URL in the dashboard">
    Navigate to **Developers** > **Webhooks** in the Grain dashboard. Select **Add Endpoint** and enter your webhook URL (for example, `https://yoursite.com/webhooks/grain`).
  </Step>

  <Step title="Select the events you want to receive">
    Choose which event types to subscribe to. You can subscribe to all events or select specific ones. It is best to subscribe only to the events your application needs.
  </Step>

  <Step title="Save and test">
    Select **Save**. Then select **Send Test Event** to send a sample payload to your endpoint and verify that it responds correctly.
  </Step>
</Steps>

## Available Webhook Events

### Payment Events

| Event                    | Description                                      |
|--------------------------|--------------------------------------------------|
| `payment.created`        | A new payment has been created                   |
| `payment.pending`        | The customer has initiated the crypto transfer    |
| `payment.completed`      | The payment has been confirmed on-chain          |
| `payment.failed`         | The payment failed or was rejected               |
| `payment.expired`        | The payment window expired without a transfer    |
| `payment.underpaid`      | The received amount is less than the required amount |

### Refund Events

| Event                    | Description                                      |
|--------------------------|--------------------------------------------------|
| `refund.created`         | A refund has been initiated                      |
| `refund.completed`       | The refund has been sent to the customer          |
| `refund.failed`          | The refund could not be processed                |

### Payout Events

| Event                    | Description                                      |
|--------------------------|--------------------------------------------------|
| `payout.created`         | A payout to your settlement account was initiated |
| `payout.completed`       | The payout has arrived in your bank account       |
| `payout.failed`          | The payout could not be completed                |

## Verify Webhook Signatures

Every webhook request includes a `Grain-Signature` header that you should verify to confirm the request came from Grain and was not tampered with.

Your webhook signing secret is available on the **Webhooks** page in the dashboard.

```javascript
const crypto = require("crypto");

function verifyWebhookSignature(payload, signature, secret) {
  const expectedSignature = crypto
    .createHmac("sha256", secret)
    .update(payload)
    .digest("hex");

  return crypto.timingSafeEqual(
    Buffer.from(signature),
    Buffer.from(expectedSignature)
  );
}
```

<Warning>
  Always verify the `Grain-Signature` header before processing a webhook event. Skipping signature verification exposes your application to spoofed requests.
</Warning>

## Retry Behavior

<Note>
  If your endpoint does not return a `200` status code, Grain retries the delivery using an exponential backoff schedule: **1 minute, 5 minutes, 30 minutes, 2 hours, and 24 hours**. After 5 failed attempts, the event is marked as failed and no further retries are made. You can manually retry any failed event from the dashboard.
</Note>

You can view the delivery status and response codes for every webhook event in **Developers** > **Webhooks** > **Event Log**.
