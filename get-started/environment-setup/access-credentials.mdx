## Retrieve Your API Keys

Your API keys authenticate every request you make to the Grain API. Follow these steps to find and manage your credentials.

<Steps>
  <Step title="Log in to the Grain Dashboard">
    Navigate to [dashboard.grain.com](https://dashboard.grain.com) and sign in with your account credentials.
  </Step>

  <Step title="Open the API Keys page">
    From the left sidebar, select **Developers** > **API Keys**. You will see your available key pairs listed by environment.
  </Step>

  <Step title="Copy your keys">
    Each environment (sandbox and production) has two keys:

    - **Public key**: Used on the client side to initialize the Grain payment widget. Starts with `pk_sandbox_` or `pk_live_`.
    - **Secret key**: Used on the server side to create payments, issue refunds, and manage resources. Starts with `sk_sandbox_` or `sk_live_`.

    Select the copy icon next to the key you need. Secret keys are partially masked by default — select **Reveal** to view the full key.
  </Step>

  <Step title="Store your keys securely">
    Add your keys to your application's environment variables or secrets manager. Never hardcode keys directly in your source code.
  </Step>
</Steps>

<Warning>
  Your secret key grants full access to your Grain account. Never expose it in client-side code, public repositories, or logs. If you believe a secret key has been compromised, rotate it immediately from the API Keys page.
</Warning>

## Sandbox vs Production Keys

| Property            | Sandbox                          | Production                        |
|---------------------|----------------------------------|-----------------------------------|
| Key prefix          | `pk_sandbox_` / `sk_sandbox_`    | `pk_live_` / `sk_live_`          |
| Availability        | Immediately after signup         | After KYB approval               |
| Processes real funds| No                               | Yes                               |
| Rate limits         | 100 requests per minute          | 1,000 requests per minute        |
| Webhook events      | Simulated                        | Real                              |

<Tip>
  Use environment variables to switch between sandbox and production keys without changing your code. For example, set `GRAIN_SECRET_KEY` in your deployment configuration.
</Tip>

## Rotating Keys

If you need to rotate a key:

1. Navigate to **Developers** > **API Keys**
2. Select **Rotate** next to the key you want to replace
3. A new key is generated immediately — the old key remains active for **24 hours** to give you time to update your integration
4. Update your application with the new key
5. After the grace period, the old key is permanently deactivated

<Note>
  Rotating a key does not affect the other key in the pair. Rotating your secret key does not change your public key, and vice versa.
</Note>
