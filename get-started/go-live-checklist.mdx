## Before You Go Live

Switching from sandbox to production means your integration will process real customer funds. Work through each item below to confirm everything is ready.

<Steps>
  <Step title="KYB verification approved">
    Confirm your KYB status shows **Approved** in **Settings** > **Verification**. You cannot process live payments until verification is complete.

    If your status is still under review, check the [KYB Dashboard](/get-started/onboarding/kyb-dashboard) for any outstanding requests.
  </Step>

  <Step title="Production API keys configured">
    Replace your sandbox keys with production keys in your deployment environment. Production keys start with `pk_live_` and `sk_live_`.

    - Store keys in environment variables or a secrets manager — never in source code
    - Update your API base URL from `api.sandbox.grain.com` to `api.grain.com`
    - Verify that your application uses the correct keys by making a test API call (for example, `GET /v1/account`)

    <Note>
      Double-check that no sandbox keys remain in your production configuration. Requests made with sandbox keys against the production API will be rejected.
    </Note>
  </Step>

  <Step title="Webhooks configured for production">
    Register your production webhook endpoint in **Developers** > **Webhooks**. Ensure you:

    - Use an HTTPS URL (HTTP endpoints are not supported in production)
    - Subscribe to the events your application needs (`payment.completed` at minimum)
    - Implement [signature verification](/get-started/environment-setup/configure-webhooks#verify-webhook-signatures) on your endpoint
    - Send a test event from the dashboard to confirm delivery
  </Step>

  <Step title="Error handling tested">
    Verify that your integration handles the following scenarios gracefully:

    - **Payment expired**: The customer did not complete the payment in time
    - **Underpayment**: The customer sent less than the required amount
    - **Network errors**: Your server lost connectivity during a payment flow
    - **Webhook delivery failure**: Your endpoint was temporarily unavailable
    - **Duplicate webhooks**: The same event was delivered more than once

    <Tip>
      Use the [sandbox environment](/get-started/environment-setup/sandbox-environment) with special test amounts to simulate each of these scenarios before going live.
    </Tip>
  </Step>

  <Step title="Settlement account verified">
    Confirm your bank account details are correct in **Settings** > **Settlements**. Grain will send payouts to this account on your configured settlement schedule.

    - Verify the account holder name matches your business registration
    - Confirm the bank account number and routing information
    - Choose your preferred settlement frequency (daily, weekly, or manual)
  </Step>

  <Step title="Customer-facing checkout reviewed">
    Walk through the full payment experience from your customer's perspective:

    - Create a payment and open the checkout URL
    - Verify your business name and branding appear correctly
    - Confirm the supported cryptocurrencies match your expectations
    - Check that success and failure redirects point to the correct pages on your site
  </Step>
</Steps>

## Go-Live Summary

Use this table as a quick reference to confirm each item is complete.

| Item                          | Where to check                          | Status |
|-------------------------------|-----------------------------------------|--------|
| KYB approved                  | Settings > Verification                 |        |
| Production keys deployed      | Your deployment environment              |        |
| API base URL updated          | Your application configuration           |        |
| Webhooks registered           | Developers > Webhooks                   |        |
| Signature verification active | Your webhook handler code                |        |
| Error scenarios tested        | Sandbox test results                     |        |
| Settlement account confirmed  | Settings > Settlements                  |        |
| Checkout experience reviewed  | Customer-facing payment flow             |        |

<Note>
  After going live, monitor your first few transactions closely in the dashboard. If anything looks unexpected, you can switch back to sandbox mode at any time by updating your API keys without affecting live data.
</Note>

## Need Help?

If you have questions before going live, contact the Grain support team at **support@grain.com** or use the live chat in the dashboard. For high-volume merchants, your dedicated account manager can schedule a pre-launch review.
